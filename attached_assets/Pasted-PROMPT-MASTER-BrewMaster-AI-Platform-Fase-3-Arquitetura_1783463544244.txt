PROMPT MASTER – BrewMaster AI Platform (Fase 3 – Arquitetura IoT e Integração com ESP32)

Você é um Arquiteto de IoT, Engenheiro de Automação Industrial, Desenvolvedor Full Stack Sênior, Especialista em ESP32, MQTT, SCADA e DevOps.

Sua missão é continuar o desenvolvimento da plataforma BrewMaster AI utilizando integralmente a arquitetura existente. Não recrie funcionalidades já implementadas. Evolua o projeto para prepará-lo para integração com hardware real.

Objetivo

Implementar toda a infraestrutura de comunicação entre o supervisório e múltiplos controladores ESP32, mantendo inicialmente um ambiente híbrido com dispositivos simulados e reais.

Requisitos Gerais

- Manter toda a arquitetura existente.
- Não remover funcionalidades implementadas.
- Seguir Clean Architecture, SOLID e TypeScript.
- Todo código deve ser modular, documentado e escalável.

Comunicação IoT

Implementar suporte aos seguintes protocolos:

- MQTT (principal)
- WebSocket
- REST API
- OTA (Over-the-Air)

Criar uma camada de abstração para que o frontend não dependa diretamente do protocolo utilizado.

Broker MQTT

Criar um módulo completo de gerenciamento MQTT.

Cada ESP32 deverá possuir:

- Device ID
- Nome
- Localização
- Modelo
- Versão de firmware
- Número de série
- Status Online/Offline
- Último Heartbeat
- Intensidade do sinal Wi-Fi
- Endereço IP
- Tempo de atividade (Uptime)

Definir uma estrutura padronizada de tópicos MQTT, por exemplo:

- brewmaster/{deviceId}/status
- brewmaster/{deviceId}/telemetry
- brewmaster/{deviceId}/commands
- brewmaster/{deviceId}/alarms
- brewmaster/{deviceId}/events
- brewmaster/{deviceId}/firmware
- brewmaster/{deviceId}/logs

Cadastro de Dispositivos

Criar um módulo completo para gerenciamento dos ESP32.

Permitir:

- Adicionar dispositivo
- Editar
- Excluir
- Agrupar por planta
- Agrupar por setor
- Visualizar mapa de dispositivos
- Histórico de conexões
- Status em tempo real

Simulador de ESP32

Enquanto o firmware não estiver pronto, criar um simulador que publique dados realistas via MQTT.

Simular:

- Temperatura
- Pressão
- pH
- Vazão
- Volume
- Densidade
- Estado das bombas
- Estado das válvulas
- Estado das resistências
- Alarmes
- Falhas
- Oscilações de sensores
- Perda de comunicação
- Reconexão automática

O simulador deverá permitir controlar velocidade e frequência das simulações.

Registrar todos os comandos enviados e suas respostas.
